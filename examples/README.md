# Examples

## [`saas-feature-launch/`](saas-feature-launch/)

**Adaptor:** `saas-feature`
**The decision:** Pricing model for a new SaaS feature (per-seat / per-workspace / per-source / flat-rate)

This is the canonical example. Every artefact the chain produces is here:
- `_shared-context.md` — the project context the chain ran on
- `brain.md` — full append-only research log
- `strategy/integration-catalog.md` — synthesised catalogue
- `strategy/decisions.md` — architecture decision records
- `strategy/tasks.json` — importable task list
- `strategy/risk-register.md` — trackable risks
- `strategy/wave-briefings/` — execution-ready briefings

If you want to understand what the pattern produces, **start here**.
